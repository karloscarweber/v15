Design Decisions

Kow.fm v15 - K is designed with automatic responsitivity in mind. I use a dynamic type scale that I picked up from these folks. It's been built out in a continuous development on my blog, kow.fm. I really like redesigns. This one gave me the opportunity to smooth out the rough edges of years of half baked redesigns.

Dynamic type scales

Kow.fm uses a dynamic type scale. What does that mean? It means that the type scale changes based on the screen size. On a small screen the base typeface is only 16px, but on a large screen it's 20px. Rather than using mediaqueries to jump between these two values, the theme adjusts the font size relative to the screen width. Small screens get smaller type, large screens get larger type. We stop scaling the steps at a certain cutoff size, in our case 1152px. Because the theme interpolates type between two sizes we also interplate margins and padding. Let's not get too far ahead though. A type scale defines the relationship between different type sizes. It's pleasing to see each type size, at each step change in a predictable manner. See figure 1 for clarity.

Using the typescale is pretty easy with some utility classes that I've got lying around. 


  .f-step--2 { font-size: var(--step--2); }
  .f-step--1 { font-size: var(--step--1); }
  .f-step-0  { font-size: var(--step-0);  }
  .f-step-1  { font-size: var(--step-1);  }
  .f-step-2  { font-size: var(--step-2);  }
  .f-step-3  { font-size: var(--step-3);  }
  .f-step-4  { font-size: var(--step-4);  }
  .f-step-5  { font-size: var(--step-5);  }

Of course this is a ghost theme, which means that we've got a .content class wrapper for everything that comes out of the wysywig, so all regular content will automatically have good styles.
